# PC弹窗轮询分页增删场景需求分析

报告模式：纯需求

Schema Version: 2.0.0

Rule Version: 2.12.0

## 本次分析范围

- PC端清仓股弹窗、雷达预选股弹窗。
- 以当前页为观察点，验证全局末尾新增，以及前一页、当前页、后一页、最后一页删除后的页面现象。
- 覆盖直接影响用户可观察轮询行为的参数冲突、rowKeyType 行分类、非轮询动态列兼容和两个原入口回归。
- 排除移动端、其他入口、StarRocks 内部实现、后端函数内部计算、审计告警和运维监控。

## 需求理解

本次验收以轮询期间数据行新增或删除后的分页同步为主线，同时覆盖会直接改变页面结果的参数冲突、rowKeyType 分类和动态列兼容。分页用例以pageSize=10的有序数据为基线，明确每次变化前所在页、变化发生页、变化后的行集合、补位来源、当前页码、总条数、总页数和全局序号。

## 规则拆解

1. FACT-001：仅两个PC弹窗纳入范围，移动端排除。
2. FACT-002：新增数据统一插入全局末尾，并同步总条数、总页数和全局序号。
3. FACT-003：删除数据后，其后的数据按全局顺序前移；若当前页不是末页，则由后一页首行逐页补位。
4. FACT-004：当前最后一页被删空时，自动回退到最近有数据的分页。
5. FACT-005：当前页大于实际总页数时，当前页重置为实际总页数并显示最近有数据页。
6. FACT-006：开始日期和结束日期均为当天且处于盘中时启用轮询；任一日期非当天或不处于盘中时不启用。
7. FACT-007：轮询增删后，汇总条与股票列表保持同一全局业务统计范围。
8. FACT-008：roundRobin、rowKey、roundRobinRowKey、visibleStartIdx、visibleEndIdx 及三类冲突参数规则来自正式方案。
9. FACT-009：rowKeyType=0/1/2/3 分别表示可视有效行、补全行、新增行和删除行。
10. FACT-010：轮询模式动态列返回所有字段，非轮询模式继续兼容动态列参数。
11. FACT-011：盘中状态动态切换规则缺失，不进入确定性 TC。
12. FACT-012：缺少两个弹窗对应的业务表、稳定 rowKey 字段和排序字段，SQL 对账保持 blocked。

## 风险点

- RISK-001（P0，来源 FACT-002 / REQ002）：当前非末页时新增导致当前页或相邻页异常跳动。
- RISK-002（P0，来源 FACT-002 / REQ002）：末页未满或已满时新增落页、总页数或序号错误。
- RISK-003（P0，来源 FACT-003 / REQ003）：当前页删除后未由后一页首行补位，或跨页出现重复漏行。
- RISK-004（P0，来源 FACT-003 / REQ003）：前一页删除后当前页未整体前移，边界行错误。
- RISK-005（P1，来源 FACT-003 / REQ003）：后一页或远端末页删除错误改变当前页。
- RISK-006（P0，来源 FACT-004 / REQ004）：当前末页部分删除或全部删除后的留页和回退错误。
- RISK-007（P0，来源 FACT-004、FACT-005 / REQ004）：第100页连续删除末两页后页码未重置或点击上一页为空。
- RISK-008（P0，来源 FACT-006 / REQ005）：开始日期、结束日期或盘中条件不满足时错误启用轮询，或全部满足时未启用。
- RISK-009（P0，来源 FACT-007 / REQ006）：轮询增删后汇总条与全局列表统计对象不一致。
- RISK-010（P0，来源 FACT-008 / REQ007）：rowKey、roundRobinRowKey 与 roundRobin 冲突组合未被拒绝。
- RISK-011（P0，来源 FACT-009 / REQ007）：rowKeyType=0/1/2/3 的有效、补全、新增、删除语义识别错误。
- RISK-012（P1，来源 FACT-010 / REQ008）：轮询返回所有字段的规则破坏非轮询动态列兼容。
- RISK-013（P1，来源 FACT-001、FACT-010 / REQ008）：两个原入口关闭轮询后的查询、翻页或动态列能力回归。

## 条件矩阵

- 采用 `grouped_cross_product`，按八类行为分组：轮询启停、新增分页、删除补位、连续删页与页码重置、汇总一致性、参数异常、rowKeyType 数据、兼容与回归。
- 共 60 个 required combination、0 个 excluded combination，全部具有 behavior 类型定位；每条均定位到具体 TC、branch_id、step_index、expected_index 和 observable_result。
- TC001-TC003 分别定位非末页新增、末页未满新增、末页已满新增；TC004-TC009 分别定位当前页、前一页、后一页、远端末页、当前末页部分删除和当前末页整页删除；TC010 独立定位第100页连续删除末两页后的三项页码行为。
- TC018 的三类冲突参数分别定位到独立步骤和预期；TC019 的 rowKeyType=0/1/2/3 分别定位到独立步骤和预期；TC020、TC021 分别定位非轮询动态列兼容和原入口非轮询能力回归。

## 测试点摘要

- TC001 / RISK-001（P0）：当前非末页，全局末尾新增。
- TC002 / RISK-002（P0）：当前末页未满时新增。
- TC003 / RISK-002（P0）：当前末页已满时新增并生成新页。
- TC004 / RISK-003（P0）：删除当前非末页数据，由后一页首行补位。
- TC005 / RISK-004（P0）：删除当前页的前一页数据，当前页整体前移。
- TC006 / RISK-005（P1）：删除当前页的后一页数据，当前页不变。
- TC007 / RISK-005（P1）：当前非末页时删除远端末页部分或全部数据。
- TC008 / RISK-006（P0）：当前最后一页部分删除，仍停留本页。
- TC009 / RISK-006（P0）：当前最后一页全部删除，回退前一页。
- TC010 / RISK-007（P0）：100页数据连续删除末两页，回退第98页且上一页非空。
- TC011-TC014 / RISK-008：当日盘中启用；开始或结束日期非当天、当日非盘中不启用。
- TC015-TC017 / RISK-009：新增、删除和同轮更新后汇总条与全局列表范围一致。
- TC018 / RISK-010：三类冲突参数组合返回明确异常。
- TC019 / RISK-011：rowKeyType=0/1/2/3 驱动有效、补全、新增和删除行识别。
- TC020 / RISK-012：非轮询动态列参数继续生效。
- TC021 / RISK-013：两个原入口的非轮询查询、翻页和动态列能力回归。

## 回归范围

- 核心回归：新增全局末尾、当前页删除补位、前一页删除位移、当前末页部分/全部删除、连续删两页回退。
- 关联回归：后一页和远端末页删除时当前页不受影响。
- 参数与兼容回归：三类冲突参数、rowKeyType=0/1/2/3、非轮询动态列及两个原入口查询和翻页能力。
- 每个分页场景检查当前页、必要的前页/后页、rowTotal、pageTotal、当前页码和全局序号。
- 不覆盖移动端、其他入口、StarRocks 内部实现、后端函数内部计算、导出、权限、审计告警和运维监控。

## 数据影响分析

新增和删除会改变跨页行集合、页边界、全局序号、rowTotal、pageTotal和当前页码。主要Oracle采用受控有序数据，直接比较变化前后的每页行集合。

## 数据验证结论

数据验证为required。执行时按用例准备固定行数和顺序，记录每行稳定标识，轮询后逐页核对集合与顺序，避免只观察当前页而遗漏跨页重复或漏行。

## 指标口径

本次不验证具体业务指标公式；验证行集合、分页、序号，以及汇总条与全局列表是否使用同一统计对象范围，不虚构具体汇总字段或公式。

## 验证 SQL 计划

状态：blocked。仅关联 FACT-012 / CONF-002；缺少两个弹窗的数据表、稳定 rowKey 字段和排序字段，暂不生成SQL。该阻塞不关联 FACT-011 / CONF-001。

## 直接对数方案

使用pageSize=10的受控有序行集合，新增或删除指定序号行后，按用例列明的前页、当前页、后页期望集合逐一核对。

## DDL 或字段补充需求

如需SQL对账，补充两个弹窗对应业务表、稳定 rowKey 字段和排序字段。
