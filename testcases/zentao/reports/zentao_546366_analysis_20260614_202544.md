# 禅道需求 546366 测试分析报告

## 基本信息

- 需求编号：546366
- 需求标题：【客户投诉报表】首页新增页签：异常反馈
- 来源类型：禅道 MCP
- 分析范围：默认分析需求文档 2.2「产品方案及规则」
- 生成时间：2026-06-14 20:25:44
- XMind 用例文件：testcases/zentao/xmind/zentao_546366_xmind_20260614_202544.md

## 需求理解

当前客户投诉异常报表的客户热线来源数据，只覆盖呼叫中心/后台受理部已创建客诉工单的数据；未创建客诉工单但内部存在异常的数据也需要纳入部门协同优化范围。本次需求在客户投诉报表首页新增一级页签「异常反馈」，把此类异常反馈数据同步到客户投诉报表中展示。

本次变更属于报表、列表、统计、查询、下钻、导出类需求，核心风险集中在新页签入口、统计时间筛选、责任归属筛选、分类统计口径、详情跳转口径、导出口径、数据更新时间展示、字典映射和历史客诉首页兼容。

## 规则拆解

### 页面入口和结构

- 客户投诉报表首页新增一级页签：客户投诉、异常反馈。
- 现有客诉首页调整为「客户投诉」页签。
- 新增「异常反馈」页签承载本次异常反馈统计列表。

### 查询条件

- 统计时间支持按天、按月，样式和时间范围限制按照当前客诉保持一致。
- 责任归属为下拉选择框，取值来自问题归属字典，支持单选和多选，默认空。

### 列表展示

- 业务场景展示协助单、客诉单。
- 反馈总量为单据总量，高亮且支持跳转详情页。
- 各分类总量和分类量展示对应工单总量，高亮且支持跳转详情页。
- 分类包括：
  - 时效类：时效内催派、超时催派、时效内催取、超时催取
  - 质量类：货物调包、未收到货、货物丢失、回单异常
  - 破损类：货物破损、变形打湿
  - 虚假类：虚假货好时间、虚假派送时间、虚假联系、未派原因不符、虚假签到
  - 费用类：运费不满、理赔费用、增值费
  - 服务类：客诉态度、司机态度、销售态度
  - 其他类：司机安全、安全反馈/举报

### 导出和更新时间

- 顶部菜单栏展示数据更新时间。
- 列表支持导出。

### 数据来源和限定口径

- 数据表/视图：vdm_callcenter.v_o_call_return_visit
- 关键字段：
  - business_code：业务编码
  - status：回访状态/处理状态，字典 ccms_return-visit_status
  - kts_type：业务类型/业务场景，字典 ccms_return-visit_source
  - lock_times：创建时间
  - question_one_level：问题类型，字典 ccms_return_type
  - question_second_level：类型二级/二级分类，由一级字典衍生
  - problem_attribution：问题归属，字典 ccms_return_attribution
  - remark：协助原因
  - customer_level：客户画像
- 限定取值：
  - status = 3，已完结
  - kts_type = 2、8，协助单、客诉单
  - question_one_level = 20、30、540、50、60、70、130，对应时效类、质量类、服务类、虚假类、费用类、破损类、其他类

## 待确认点

- 需求描述中的默认范围规则为「第三点产品实现方案、规则」，但本需求正文实际规则位于 2.2「产品方案及规则」，本报告按 2.2 执行。
- 统计时间按天、按月的默认值、最大可选跨度、跨月/跨年限制、起止边界是否含当天需确认。
- 责任归属下拉值清单、是否支持全选/清空、搜索、无归属数据展示方式需确认。
- 反馈总量、分类总量、分类量之间的汇总关系和去重口径需确认，尤其同一 business_code 多条记录是否计一次。
- question_second_level 的具体二级分类枚举值及其与页面列名的映射关系需确认。
- 高亮跳转详情页的页面入口、详情字段、筛选条件继承、返回后状态保持需确认。
- 数据更新时间来源、刷新频率、失败/延迟时展示方式需确认。
- 导出字段范围、字段顺序、文件命名、最大导出量、是否异步导出需确认。
- 权限和安全配置章节为空，异常反馈页签、详情页、导出权限和组织数据范围需确认。
- 空数据、异常状态、历史数据兼容、上游字段缺失处理规则需确认。

## 测试点

### P0 核心测试点

- 客户投诉报表首页一级页签正确调整，原有客户投诉能力不丢失。
- 异常反馈页签按 status、kts_type、question_one_level 限定口径统计。
- 统计时间按天/按月筛选结果准确，时间边界和原客诉限制一致。
- 责任归属单选、多选、为空时的筛选口径准确。
- 反馈总量、各一级分类总量、二级分类量与明细数据一致。
- 高亮数字跳转详情页后，详情范围与点击的统计单元格一致。
- 导出结果与当前筛选、当前统计口径一致。
- 数据更新时间展示准确，能反映本页数据刷新状态。

### P1 重点测试点

- 业务场景协助单、客诉单字典映射正确。
- 一级分类、二级分类字段映射正确，未纳入限定范围的数据不展示。
- 空数据、上游字段为空、字典值缺失时展示和统计稳定。
- 多条件组合筛选后，列表、详情、导出三处口径一致。
- 历史客户投诉页签入口、查询、列表、导出不受新增页签影响。

### P2 补充测试点

- 大数据量查询和导出性能稳定。
- 页面刷新、切换页签、切换按天/按月后条件状态符合产品预期。
- 导出字段名称、顺序、日期格式、数字格式符合页面和需求。

## 回归范围

- 客户投诉报表首页原有「客户投诉」页签。
- 客户投诉报表原有统计时间查询、时间范围限制、列表展示和导出。
- 客户投诉报表详情页/下钻页的筛选条件继承和返回链路。
- 与 vdm_callcenter.v_o_call_return_visit 同源数据相关的异常反馈统计口径。
- 字典 ccms_return-visit_status、ccms_return-visit_source、ccms_return_type、ccms_return_attribution 的映射展示。
- 报表顶部数据更新时间展示公共组件或公共逻辑。

## 风险点

- 新页签承接未创建客诉工单的数据，若数据过滤仍复用原客诉口径，可能漏统。
- 分类列较多，一级/二级分类映射错误容易导致统计错列或漏列。
- 反馈总量与分类量如果未明确去重口径，容易出现汇总不等于明细。
- 详情跳转若未继承统计时间、责任归属、业务场景、分类条件，容易造成页面统计和详情不一致。
- 导出若复用旧客诉导出口径，可能缺少异常反馈分类字段或筛选条件。
- 权限章节为空，若默认开放新页签或导出，存在越权访问风险。
