# 禅道需求分析报告：正式/模拟交易下转K线参数优化

## 一、本次分析范围

- 来源类型：用户手动粘贴禅道需求内容
- 需求标题：【正式/模拟交易】下转K线-参数优化：汇总卡片跳转K线需要新增参数“标识”+“资金源组”
- 涉及模块：
  - 正式交易-临底
  - 模拟交易-临底
- 分析范围：需求第三部分“产品实现方案、规则”为主，结合背景、注意点、异常说明作为补充参考。
- 输出模式：完整输出流程中的第一阶段分析。由于当前存在待确认点，按规则暂不生成最终 XMind Markdown 用例。

## 二、需求理解

当前正式交易跳转 K 线图后，K 线图页面的数据标点不会实时更新。策略员期望在当前股票及资金源组维度下实时更新标点情况，因此汇总卡片跳转 K 线时，需要在既有跳转参数基础上新增：

- 标识：默认传 `1`，部分历史交易日场景不传。
- 资金源组：根据首页查询条件和弹窗查询条件的资金源组筛选状态计算，满足条件时随跳转参数传递。

本次改动属于“汇总卡片/弹窗下钻跳转参数优化”，核心验证目标不是页面展示新增字段，而是不同入口、不同指标、不同交易日期、不同资金源组筛选组合下，跳转到 K 线的参数是否符合规则，并且正式交易和模拟交易均一致覆盖。

## 三、规则拆解

### 3.1 页面入口与字段

涉及页面路径：

- 汇总卡-T0股-弹窗
- 汇总卡-T0股-底仓-弹窗

涉及跳转字段/指标：

- 最新涨幅
- 交易日涨幅
- 底仓浮盈
- T0盈率
- T0合盈率

### 3.2 通用参数规则

- 策略编码：
  - 多数指标描述为“T0底仓的所有策略（包含建仓和T0策略）+列表所有策略”。
- 策略成员ID：
  - 最新涨幅、底仓浮盈、T0盈率描述为“T0底仓的所有策略（包含建仓和T0策略）+列表所有策略”的策略成员。
  - 交易日涨幅描述为“T0策略+列表所有策略”的策略成员，与策略编码口径存在差异，需要确认是否为产品有意设计。
- 标识：
  - 默认传 `1`。
  - 交易日涨幅：如果交易日期不等于股市最新交易日，标识和资金源组都不传。
  - T0盈率：如果交易日期不等于股市最新交易日，标识和资金源组都不传。
- 资金源组：
  - 首页查询条件“资金源组”和弹窗内查询条件“资金源组”都有筛选：取交集。
  - 首页查询条件“资金源组”和弹窗内查询条件“资金源组”都未筛选：不传资金源组。
  - 首页查询条件“资金源组”有值，弹窗内查询条件“资金源组”无值：取首页查询条件“资金源组”。
  - 首页查询条件“资金源组”无值，弹窗内查询条件“资金源组”有值：取弹窗查询条件“资金源组”。
  - 如果资金源组选择后未点击查询，则认为查询无效，资金源组按空处理。
  - 资金源组下转后，再点击跳转时，也需要添加这些参数。

### 3.3 指标差异规则

| 字段名称 | 交易日期规则 | 标识规则 | 资金源组规则 |
| --- | --- | --- | --- |
| 最新涨幅 | 股市最新交易日期 | 默认传 `1` | 按首页与弹窗资金源组筛选规则计算 |
| 交易日涨幅 | 交易日涨幅的交易日期 | 最新交易日传 `1`；非最新交易日不传标识和资金源组 | 最新交易日按规则计算；非最新交易日不传 |
| 底仓浮盈 | 股市内的最新交易日 | 默认传 `1` | 按首页与弹窗资金源组筛选规则计算 |
| T0盈率 | 查询内T0策略的最大交易日期 | 最新交易日传 `1`；非最新交易日不传标识和资金源组 | 最新交易日按规则计算；非最新交易日不传 |
| T0合盈率 | 需求仅列字段名，未描述交易日期、策略、成员、标识、资金源组规则 | 待确认 | 待确认 |

## 四、初步影响范围

- 正式交易-临底汇总卡跳转 K 线。
- 模拟交易-临底汇总卡跳转 K 线。
- 汇总卡 T0 股弹窗与 T0 股底仓弹窗内的多个指标下钻。
- K 线页面接收参数后的标点实时刷新逻辑。
- 首页资金源组筛选、弹窗资金源组筛选、资金源组下转后的二次跳转链路。
- 历史交易日与最新交易日的参数兼容逻辑。

## 五、待确认点

1. T0合盈率是否也需要新增“标识”和“资金源组”参数，以及交易日期、策略编码、策略成员ID口径是否与 T0盈率一致。
   - 影响：若不确认，T0合盈率无法生成确定性跳转参数预期，可能遗漏一个核心指标链路。
   - 建议确认：明确 T0合盈率的交易日期取值、标识传递条件、资金源组传递条件、策略编码和策略成员ID口径。

2. 首页与弹窗资金源组都筛选时“取交集”，如果交集为空，跳转时是“不传资金源组”、传空数组/空字符串，还是阻断跳转/提示无数据。
   - 影响：交集为空是高频边界场景，不同处理会直接影响 K 线标点范围和接口参数。
   - 建议确认：明确交集为空时的参数形态和页面行为。

3. “资金源组选择后未点击查询，则认为查询无效”适用于首页筛选、弹窗筛选，还是两者都适用。
   - 影响：若首页和弹窗处理不一致，会影响资金源组取值来源和用例预期。
   - 建议确认：明确未点击查询的判定范围，以及重置/切换/关闭弹窗后是否保留已生效条件。

4. “资金源组下转后，再点击跳转时，也需要添加这些参数”中的“下转后”具体指哪一级页面或哪种操作链路。
   - 影响：会影响回归范围和链路型用例设计，尤其是从资金源组维度进入后再次跳 K 线的参数继承。
   - 建议确认：给出资金源组下转页面路径、二次跳转入口和参数继承口径。

5. 新增参数“标识”和“资金源组”的真实参数名、编码格式、分隔符、多选顺序、是否传 code 或 name 未在需求中说明。
   - 影响：接口/URL 参数验收无法写成确定性预期，尤其多选资金源组和交集结果的断言依赖参数格式。
   - 建议确认：明确 URL 参数名、资金源组值格式、是否去重、是否保持选择顺序或按后端标准排序。

6. “股市最新交易日”“交易日涨幅的交易日期”“查询内T0策略的最大交易日期”的数据源未说明。
   - 影响：最新交易日判断是是否传标识和资金源组的关键条件，数据源不明会导致边界验证口径不一致。
   - 建议确认：明确三个交易日期的来源接口/字段或业务口径。

7. 正式交易与模拟交易是否完全复用同一套参数规则，是否存在策略编码、资金源组、交易日来源差异。
   - 影响：若两端数据源不同，需要分别设计断言；若完全一致，可复用用例并做双模块覆盖。
   - 建议确认：明确正式/模拟交易是否存在差异规则。

8. K 线页面接收“标识=1 + 资金源组”后，标点实时更新的触发方式和刷新频率未说明。
   - 影响：本需求背景关注实时更新，但实现规则只说明跳转参数，是否需要验证刷新机制尚不明确。
   - 建议确认：本次验收是否只校验跳转参数，还是还要校验 K 线标点实时刷新结果。

## 六、风险点

- T0合盈率规则缺失，可能导致该指标漏传新增参数或沿用错误口径。
- 交易日涨幅和 T0盈率在非最新交易日不传标识和资金源组，容易出现只漏判其中一个参数的缺陷。
- 首页与弹窗资金源组交集、多选、未点击查询等状态组合复杂，容易出现使用未生效筛选值或未取交集的问题。
- 策略编码与策略成员ID取值口径包含“T0底仓策略、建仓策略、T0策略、列表策略”等组合，容易漏策略、重复策略或策略成员ID口径不一致。
- 正式交易和模拟交易双模块需要一致覆盖，容易只改一端。
- 二次跳转链路如果未继承新增参数，会导致首次跳转正确但下转后再次跳转丢参。
- 参数格式未明确，研发和测试可能对资金源组 code/name、多选拼接方式理解不一致。
- 如果 K 线页面存在旧参数兼容逻辑，新增参数可能影响历史交易日、无资金源组、无数据场景的原有跳转。

## 七、测试点摘要

### P0 核心测试点

- 正式交易-临底：最新涨幅、底仓浮盈在最新交易日场景跳 K 线时传 `标识=1`，资金源组按筛选规则计算。
- 模拟交易-临底：同正式交易规则覆盖，确保双模块一致。
- 交易日涨幅：交易日期等于股市最新交易日时传 `标识=1` 和有效资金源组；不等于最新交易日时标识和资金源组都不传。
- T0盈率：查询内 T0 策略最大交易日期等于股市最新交易日时传新增参数；不等于最新交易日时不传。
- 资金源组四类组合：双侧有值取交集、双侧无值不传、仅首页有值取首页、仅弹窗有值取弹窗。
- 资金源组选择但未点击查询时按空处理，不应把未生效选择值带入 K 线跳转参数。
- 从资金源组下转后再次点击跳转 K 线，仍需追加标识和资金源组参数。
- 策略编码和策略成员ID覆盖 T0底仓、建仓、T0策略、列表策略的组合口径，避免漏传或错传。

### P1/P2 扩展测试点

- 多选资金源组包含重叠、部分重叠、完全不重叠的组合。
- 弹窗筛选后关闭再打开、重置筛选、重新查询后的资金源组生效状态。
- 首页筛选条件变更后未查询、已查询两种状态下的跳转参数。
- 无数据、空策略、空策略成员、资金源组为空时跳转兼容。
- 旧入口、历史交易日入口、未带新增参数入口仍可正常打开 K 线。
- K 线页面接收新增参数后标点范围与当前股票、资金源组范围一致。
