# SQL 与数据 Profile

只使用需求、SQL 说明、Diff 或代码中真实存在的标识和口径。

## 数据口径

- 明确数据源、主键、关联键、过滤条件、时间范围、去重键、聚合层级和排序。
- 检查一对一、一对多、多对多关联是否造成丢失或重复。
- 检查 null、空串、零、负值、极值、重复、孤儿数据和未知枚举。
- 检查 count、sum、avg、distinct、分组、窗口和汇总层级。
- 金额与比例明确精度、舍入位置、累计误差和展示格式。
- 时间明确时区、交易日、自然日、边界包含关系和夏令时。

## 查询和性能

- 检查分页稳定排序、过滤下推、索引命中、全表扫描、N+1、循环查询和大结果集。
- 导出、汇总和批处理检查超时、内存、锁、并发和数据新鲜度。
- 性能阈值无证据时只提出验证方法，不写具体数字。

## 迁移和回填

- 检查向前和向后兼容、默认值、历史数据、幂等、断点续跑、失败重试和回滚。
- 对比迁移前后记录数、主键集合、关键汇总、校验和及异常清单。
- 回填脚本检查范围、重复执行、审计记录、灰度和恢复。

## SQL 验证约束

- 报告中标明 SQL 证据来源和环境。
- 未提供表名或字段时写验证思路，不生成带虚构标识的 SQL。
- 代码查询结果只能作为实现证据，不能自动成为业务预期。
