# 权限安全 Profile

普通功能主链路可先按有权限用户验证，但这不等于权限没有影响或无需权限测试。命中用户、账号、组织、部门、角色、资金/策略账户、数据范围、行列权限、导出、下钻、详情、公共/个人配置、敏感字段、跨账号/组织或多租户时，必须输出权限风险；缺少口径则进入待确认，不得宣称“权限无影响”。

普通功能默认按有权限用户验证主链路。只有权限、安全或数据隔离会改变核心验收时才升级为阻塞问题。

## 权限模型

- 区分认证、菜单、按钮、接口、行、列、组织、角色和数据范围。
- 明确授权来源、组合关系、优先级、继承、缓存和生效时机。
- 检查同角色、跨角色、跨组织、跨账号和管理员边界。

## 越权和隔离

- 页面隐藏不能替代接口鉴权；直接调用、修改参数、批量接口和导出均需验证。
- 列表、详情、汇总、下钻、弹窗、导出和缓存必须使用同一已确认数据范围。
- 检查对象级越权、水平越权、垂直越权和历史链接失效。

## 敏感数据

- 明确脱敏字段、格式、角色差异、日志、导出、错误信息和审计。
- 不在测试产物中记录真实敏感数据、令牌或凭据。
- 未说明脱敏口径时进入待确认点，不自行设计星号格式。
