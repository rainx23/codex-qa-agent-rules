# QA 规则验收清单

## 适用范围

本文件用于验证 `codex-qa-agent-rules` 中的规则是否能被 Codex 稳定执行，适用于规则修改后自检、发布前验收和后续回归。

## 验收原则

- 只验证规则行为，不修改业务代码。
- 不要求生成真实业务测试产物，可使用模拟需求、模拟 diff 或简化示例。
- 验收时必须关注：不编造、先确认、少重复、证据可追溯、XMind 格式正确、疑似缺陷识别准确。

## 核心验收场景

### 场景 1：纯需求分析

输入：只有禅道需求或手动需求文本，没有 diff。

期望：

- 输出需求理解、规则拆解、待确认点、风险点、测试点、回归范围。
- 信息不足时先抛待确认点。
- 不生成无证据的接口、字段、SQL、页面入口。
- 不强制输出疑似缺陷章节。

### 场景 2：纯 diff 分析

输入：只有 commit 或 diff，没有需求上下文。

期望：

- 说明未提供需求基准。
- 只基于 diff 证据分析风险。
- 不反推需求事实。
- 输出 diff 文件清单、核心改动点、疑似风险点、疑似缺陷、P0 测试点、回归范围。
- 无明确疑似缺陷时输出“未发现明确疑似缺陷”。

### 场景 3：需求 + diff 一致

输入：需求规则和 diff 实现一致。

期望：

- 输出需求-Diff-测试点追踪矩阵。
- diff 覆盖情况标记为已覆盖。
- 生成对应测试点和回归范围。
- 疑似缺陷章节输出“未发现明确疑似缺陷”。

### 场景 4：需求 + diff 不一致

输入：需求明确要求字段 A 同步页面和导出，但 diff 只改页面未改导出。

期望：

- 输出需求-Diff-测试点追踪矩阵。
- 标记 diff 疑似遗漏或实现不一致。
- 输出疑似缺陷，包含需求证据、diff 证据、影响范围、建议验证方式。
- P0/P1 测试点覆盖页面与导出一致性。
- 不直接修改业务代码。

### 场景 5：证据不足

输入：需求没有说明字段来源，diff 中也无法确认字段来源。

期望：

- 输出待确认点或疑似风险点。
- 不输出确定性预期。
- 不把风险写成疑似缺陷。
- 不编造 SQL、字段、接口。

### 场景 6：无业务 diff

输入：diff 只有注释、格式化、日志文案或文档变化。

期望：

- 不强行生成业务 XMind 用例。
- 输出无需新增业务测试点或最小冒烟建议。
- 不生成低价值 TC。

### 场景 7：XMind 对话展示

输入：用户要求“给我完整 XMind Markdown，代码块格式”。

期望：

- 对话中使用 markdown 代码块包裹完整 XMind Markdown。
- 代码块只是对话展示方式。
- 不影响本地文件格式规则。

### 场景 8：本地 XMind 文件

输入：用户要求生成本地 XMind Markdown 和 `.xmind` 文件。

期望：

- 本地 `.md` 文件必须直接从 `- 根节点名称` 开始。
- 本地文件禁止代码块、解释说明、总结说明、表格、JSON。
- TC 编号从 TC001 连续递增。
- 缩进统一 4 个空格。
- 可通过 `scripts/md_to_xmind.py` 转换。

### 场景 9：最小有效用例集

输入：一个包含多个类似字段、多个类似模块的需求。

期望：

- 相同入口、相同规则、相同预期的 TC 被合并。
- 仅模块名称不同但验证逻辑相同的 TC 被合并。
- 不为了模板完整性机械生成低价值 TC。
- 保留 P0/P1 高风险场景。

### 场景 10：金融/交易/策略数据类

输入：涉及金额、数量、收益率、批次、策略、股票、账户、成交编号、汇总明细一致性的需求或 diff。

期望：

- 重点扫描数据来源、聚合粒度、去重、日期口径、精度、汇总明细一致性。
- 关注多买多卖、一个卖策略对应多个买策略、买入先取 max 再聚合等重复统计风险。
- 信息不足时只输出验证思路和待确认点，不编造表名字段 SQL。

### 场景 11：个性化配置 / 字段设置 / 快捷查询设置

输入：需求或 diff 涉及新增字段、新增快捷查询条件、个性设置、字段设置、列配置、模板配置或快捷查询设置。

期望：

- 如果是新增字段，应扫描该字段是否进入个性设置字段列表。
- 验证字段勾选后页面展示，取消勾选后页面隐藏。
- 验证字段拖动排序、输入序号移动后，页面字段顺序是否生效。
- 验证字段固定列配置后，页面横向滚动时固定列是否生效。
- 验证保存、应用、取消、重置、另存为后配置是否符合预期。
- 验证刷新页面、重新进入页面、切换模板后配置是否保持或按模板生效。
- 如果是新增快捷查询条件，应扫描该条件是否进入快捷查询设置。
- 验证快捷查询条件勾选 / 不勾选后，页面快捷查询区域是否展示或隐藏。
- 验证快捷查询条件拖动排序后，页面快捷查询条件顺序是否生效。
- 验证快捷查询条件保存应用后，查询参数和查询结果是否正确。
- 证据不足时输出待确认点，不编造默认勾选状态、模板权限、接口字段或存储方式。
- 不为每个字段机械生成重复 TC，相同配置动作和相同预期应合并。

## 发布前自检

规则修改后至少确认：

- AGENTS.md 中工作边界仍然有效。
- P0/P1/P2 体系未被破坏。
- 没有引入 P3。
- XMind 本地文件规则与对话展示规则不冲突。
- diff 输出模板、对话输出清单、本地报告内容三者一致。
- 需求 + diff 联动时能输出疑似缺陷。
- 证据不足时不会把风险升级为疑似缺陷。
- 通用模板不会被机械转成低价值 TC。
- 新增字段、新增快捷查询条件时，已扫描个性设置、字段设置、快捷查询设置、拖动排序、勾选隐藏、固定列和保存应用风险。
