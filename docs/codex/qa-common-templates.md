# QA 通用业务模板

## 适用范围

本文件提供常见业务类型的测试关注点模板。模板用于辅助分析，不代表每个需求都必须全部生成用例。

使用模板时必须遵守：

- 只选择与当前需求、diff 或风险证据相关的关注点。
- 不把模板项机械转成 TC。
- 不强制每条 TC 具备前置条件、测试数据、优先级等固定字段。
- 证据不足时只输出风险点或待确认点，不虚构字段、接口、SQL、权限或页面入口。

## 查询类

- 默认查询条件、默认时间范围、默认组织或权限范围。
- 单条件筛选、多条件组合筛选、清空条件后查询。
- 已选择但未点击查询的条件是否生效。
- 多选条件的全选、部分选择、取消选择、交集、并集和空集合。
- 枚举、字典、状态、组织、人员、日期范围、模糊搜索的有效值和无效值。
- 查询无数据、大数据量、分页后再次查询。
- 查询条件与导出、下钻、弹窗、跳转参数是否保持一致。

## 列表和表格类

- 默认列表数据、字段名称、字段顺序、字段来源、字段为空展示。
- 分页、每页条数、排序、固定列、横向滚动、+N 展开。
- 汇总行、合计行、明细行和弹窗明细的一致性。
- 列表行点击、链接点击、下钻、弹窗、返回后状态保持。
- 页面刷新、重新查询、筛选条件变更后的数据刷新。
- 不同角色、组织、数据范围下的列表数据隔离。

## 弹窗类

- 打开入口、弹窗标题、默认查询条件、默认数据范围。
- 弹窗内查询条件与外层页面查询条件的继承、交集、覆盖或互不影响规则。
- 弹窗内选择条件但未点击查询时，是否视为未生效。
- 弹窗内列表、分页、排序、+N 展开、下钻、关闭后再次打开。
- 弹窗与外层页面数据口径、权限范围、状态条件是否一致。

## 跳转和下钻类

- 跳转入口、目标页面、携带参数、参数来源、参数为空处理。
- 页面已打开后再次跳转，目标页数据是否刷新。
- 参数新增、删除、默认值、兼容旧参数。
- 列表行、汇总卡、弹窗行、+N 展开行等二次入口的参数一致性。
- 跳转前筛选条件、弹窗条件、已生效查询条件与目标页参数映射。
- 目标页返回、刷新、重复跳转、跨模块跳转。

## 导出类

- 默认条件导出、筛选后导出、排序后导出。
- 导出字段名称、字段顺序、字段值、字段格式、精度和单位。
- 页面简写值与导出完整值一致。
- 汇总、明细、弹窗、下钻导出口径一致。
- 空数据、大数据量、无权限、接口失败或超时。

## 接口类

- 入参新增、删除、类型变更、必填变更、默认值变更。
- 出参字段新增、删除、重命名、类型变更、枚举值变更。
- 返回结构层级、空值、空数组、错误码、兼容旧调用方。
- 上游调用方和下游依赖方是否同步调整。
- 接口超时、失败、重复请求、幂等和异常返回。

## SQL 和数据口径类

- where 条件、join 条件、group by 粒度、distinct 去重。
- count、sum、avg、max、min 等聚合口径。
- 日期口径、交易日口径、时区、自然日与业务日差异。
- 权限条件、组织条件、状态条件、删除标记、有效标记。
- 汇总与明细、页面与导出、接口与 SQL 结果一致。
- 信息不足时，只能写 SQL 验证思路，不编造表名字段。

## 金融/交易/策略数据类

- 金额、数量、比例、收益率、成本、盈亏字段精度。
- 汇总与明细是否一致。
- 页面、接口、导出、SQL 四方口径是否一致。
- 交易日、自然日、盘中时间、期初期末时间边界。
- 股票、策略、账户、批次、成交编号等维度是否重复或漏聚合。
- 建仓、加仓、减仓、清仓、撤单、失败、部分成交、重复成交等状态场景。
- 0 值、负值、空值、极大值、科学计数法展示。
- 多买、多卖、一个卖策略对应多个买策略、买卖关系重复统计风险。
- 买入相关数据先取最大值再聚合、卖出相关数据按真实成交聚合的口径风险。
- 权限、账户、组织、策略范围隔离。
- 历史数据、新旧口径、交易日历和非交易日兼容。
- 盘中数据、日终数据、T 日、T-1 日、期初、期末口径一致性。
- 页面汇总、弹窗明细、下钻页面、导出文件之间的数据一致性。
- 批次、股票、策略、账户、成交编号等多维度组合导致的重复统计和漏统计风险。

使用约束：

- 该模板只作为分析辅助，不代表每个金融/交易需求都必须全部生成 TC。
- 只有命中真实需求、diff、SQL 或业务风险时才生成对应测试点。
- 不得凭模板虚构表名、字段名、SQL、接口或页面入口。
- 金融/交易/策略数据类需求中，优先关注核心数据口径、汇总明细一致性、精度、权限隔离和历史兼容。
- 对金额、数量、比例、收益率、成本、盈亏等字段，必须重点扫描精度、单位、四舍五入、空值、0 值、负值和极端值风险。
- 如果需求或 diff 涉及 SQL，必须关注 where、join、group by、distinct、聚合粒度、日期口径、有效标记、权限条件。

## 权限类

- 有权限用户可查看或操作。
- 无权限用户不可查看或操作。
- 不同角色、组织、部门、数据范围隔离。
- 下钻、弹窗、导出、接口直接访问是否继承权限。
- 越权访问、参数篡改、跨组织数据访问。

## 状态和流程类

- 初始状态、处理中状态、终态、异常状态。
- 合法状态流转、非法状态流转、重复提交。
- 状态变更后的列表、详情、汇总、导出和下游同步。
- 历史数据、新旧状态、字典映射和兼容。

## 配置和开关类

- 配置开启、关闭、默认值、灰度范围。
- 配置变更后是否实时生效，是否需要刷新、重新登录或重新查询。
- 配置缺失、异常值、回滚后行为。
- 不同环境、组织、角色下配置差异。

## 日志、审计和通知类

- 关键操作是否记录日志或审计信息。
- 操作人、操作时间、操作对象、变更前后值是否可追踪。
- 通知触发条件、接收人、重复通知、失败补偿。
- 需求未说明日志或通知口径时，只列风险和待确认点，不生成确定预期。
