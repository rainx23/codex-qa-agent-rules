# QA 通用业务模板

## 适用范围

本文件提供常见业务类型的测试关注点模板。模板用于辅助分析，不代表每个需求都必须全部生成用例。

使用模板时必须遵守：

- 只选择与当前需求、diff 或风险证据相关的关注点。
- 不把模板项机械转成 TC。
- 不强制每条 TC 具备前置条件、测试数据、优先级等固定字段。
- 证据不足时只输出风险点或待确认点，不虚构字段、接口、SQL、权限或页面入口。

## 查询类

- 默认查询条件、默认时间范围、默认组织或权限范围。
- 单条件筛选、多条件组合筛选、清空条件后查询。
- 已选择但未点击查询的条件是否生效。
- 多选条件的全选、部分选择、取消选择、交集、并集和空集合。
- 枚举、字典、状态、组织、人员、日期范围、模糊搜索的有效值和无效值。
- 查询无数据、大数据量、分页后再次查询。
- 查询条件与导出、下钻、弹窗、跳转参数是否保持一致。

## 列表和表格类

- 默认列表数据、字段名称、字段顺序、字段来源、字段为空展示。
- 分页、每页条数、排序、固定列、横向滚动、+N 展开。
- 汇总行、合计行、明细行和弹窗明细的一致性。
- 列表行点击、链接点击、下钻、弹窗、返回后状态保持。
- 页面刷新、重新查询、筛选条件变更后的数据刷新。
- 不同角色、组织、数据范围下的列表数据隔离。

## 个性化配置 / 字段设置 / 快捷查询设置类

适用于页面存在“个性设置”“字段设置”“快捷查询设置”“列配置”“模板配置”“保存视图”“自定义查询”等能力时。

### 字段设置

- 字段勾选后，列表、表格、弹窗或导出中对应字段是否展示。
- 字段取消勾选后，对应字段是否隐藏，且不影响其他字段展示。
- 全选后是否展示所有可配置字段。
- 取消全选或最少勾选限制是否符合需求规则。
- 新增字段是否出现在个性设置字段列表中。
- 新增字段默认是否勾选，默认状态是否符合需求。
- 新增字段勾选保存后，页面是否展示该字段。
- 新增字段取消勾选保存后，页面是否隐藏该字段。
- 字段名称、字段顺序、字段分组、字段层级是否与页面实际展示一致。
- 字段为空、0、负数、特殊值时，个性设置隐藏/展示后页面表现是否一致。

### 字段拖动排序

- 在个性设置中上下拖动字段后，保存并应用，页面字段顺序是否同步变化。
- 拖动多个字段后，页面展示顺序是否与配置顺序一致。
- 拖动父级字段或分组字段时，子字段顺序和层级是否正确。
- 刷新页面、重新进入页面、关闭后重新打开后，字段顺序是否保持。
- 切换模板、重置、另存为后，字段顺序是否按对应模板生效。
- 拖动排序不应影响字段值、字段口径、查询结果和导出数据。

### 输入序号移动

- 输入合法序号后，字段是否移动到指定位置。
- 输入序号为 1、最后一位、中间位置时，字段顺序是否正确。
- 输入重复序号、空值、0、负数、超出范围、非数字时，系统处理是否符合需求。
- 输入序号移动后保存并应用，页面字段顺序是否生效。
- 输入序号移动和拖动排序混合使用时，最终顺序是否以最后保存配置为准。

### 固定列

- 勾选固定列后，页面横向滚动时该字段是否固定展示。
- 取消固定列后，页面横向滚动时该字段是否恢复普通列。
- 多个字段固定列时，固定顺序是否与字段配置顺序一致。
- 固定列字段被取消勾选时，页面不应继续展示该固定列。
- 固定列与字段拖动排序同时配置时，固定列位置和普通列顺序是否正确。
- 固定列配置保存、刷新、重新进入页面后是否保持。
- 固定列配置不应影响导出字段顺序，除非需求明确要求导出跟随个性化顺序。

### 保存 / 另存为 / 应用 / 重置

- 保存后，当前模板配置是否持久化。
- 应用后，页面字段展示、字段顺序、固定列、快捷查询条件是否立即生效。
- 另存为新模板后，新模板是否可搜索、可切换、可再次编辑。
- 重置后，字段勾选、字段顺序、固定列、快捷查询条件是否恢复默认。
- 取消后，未保存配置不应生效。
- 关闭弹窗后再次打开，已保存配置是否正确回显。
- 多次保存、连续修改、重复应用时，配置不应错乱。

### 模板搜索和切换

- 搜索模板名支持模糊匹配时，输入完整名称、部分名称、编号、空值时结果是否正确。
- 搜索无结果时是否有合理提示。
- 切换不同模板后，字段勾选、字段顺序、固定列、快捷查询条件是否跟随模板切换。
- 删除、重命名、另存模板后，模板列表和已应用配置是否正确。
- 默认模板、个人模板、公共模板之间的生效优先级是否符合需求。

### 快捷查询设置

- 新增快捷查询条件后，页面快捷查询区域是否展示对应条件。
- 取消勾选快捷查询条件后，页面快捷查询区域是否隐藏对应条件。
- 快捷查询条件拖动排序后，页面快捷查询条件顺序是否同步变化。
- 快捷查询条件全选、取消全选、部分勾选是否生效。
- 快捷查询条件保存并应用后，刷新页面或重新进入页面是否保持。
- 快捷查询条件与普通查询条件同时存在时，查询参数是否正确。
- 快捷查询条件值为空、默认值、多选值、日期范围、枚举值时，查询是否符合需求。
- 快捷查询配置不应影响非快捷查询区域，除非需求明确要求联动。
- 新增快捷查询条件时，应验证字段来源、默认值、枚举映射、是否必填、是否支持清空。

### 权限和数据隔离

- 不同用户的个性化配置是否互相隔离。
- 不同角色、组织、账号的模板配置是否按权限展示。
- 无权限用户是否不可编辑公共模板或他人模板。
- 切换账号、退出登录、重新登录后，个人配置是否正确恢复。
- 公共模板和个人模板同时存在时，生效优先级是否符合需求。

### 兼容和回归

- 新增字段后，历史模板是否兼容，不应导致页面报错或字段配置丢失。
- 删除或下线字段后，历史模板中对应字段是否安全忽略或按规则处理。
- 字段重命名后，历史模板是否能正确映射。
- 字段配置变更不应影响列表查询、排序、分页、导出、下钻、弹窗数据口径。
- 个性设置配置变更后，应验证页面展示、接口参数、缓存、本地存储或服务端配置是否一致。

### 使用约束

- 该模板只作为分析辅助，不代表每个需求都必须全部生成 TC。
- 只有需求、diff、页面截图或代码证据涉及个性设置、字段设置、快捷查询设置、列配置、模板配置时才生成对应测试点。
- 证据不足时只输出待确认点或风险点，不虚构字段、接口、存储方式或权限规则。
- 如果需求只说“新增字段”，但页面存在个性设置字段配置入口，应默认扫描该字段是否需要进入字段设置、是否可勾选、是否可排序、是否可固定列。
- 如果需求只说“新增快捷查询条件”，应默认扫描该条件是否需要进入快捷查询设置、是否可勾选、是否可排序、是否保存应用后生效。

## 弹窗类

- 打开入口、弹窗标题、默认查询条件、默认数据范围。
- 弹窗内查询条件与外层页面查询条件的继承、交集、覆盖或互不影响规则。
- 弹窗内选择条件但未点击查询时，是否视为未生效。
- 弹窗内列表、分页、排序、+N 展开、下钻、关闭后再次打开。
- 弹窗与外层页面数据口径、权限范围、状态条件是否一致。

## 跳转和下钻类

- 跳转入口、目标页面、携带参数、参数来源、参数为空处理。
- 页面已打开后再次跳转，目标页数据是否刷新。
- 参数新增、删除、默认值、兼容旧参数。
- 列表行、汇总卡、弹窗行、+N 展开行等二次入口的参数一致性。
- 跳转前筛选条件、弹窗条件、已生效查询条件与目标页参数映射。
- 目标页返回、刷新、重复跳转、跨模块跳转。

## 导出类

- 默认条件导出、筛选后导出、排序后导出。
- 导出字段名称、字段顺序、字段值、字段格式、精度和单位。
- 页面简写值与导出完整值一致。
- 汇总、明细、弹窗、下钻导出口径一致。
- 空数据、大数据量、无权限、接口失败或超时。

## 接口类

- 入参新增、删除、类型变更、必填变更、默认值变更。
- 出参字段新增、删除、重命名、类型变更、枚举值变更。
- 返回结构层级、空值、空数组、错误码、兼容旧调用方。
- 上游调用方和下游依赖方是否同步调整。
- 接口超时、失败、重复请求、幂等和异常返回。

## SQL 和数据口径类

- where 条件、join 条件、group by 粒度、distinct 去重。
- count、sum、avg、max、min 等聚合口径。
- 日期口径、交易日口径、时区、自然日与业务日差异。
- 权限条件、组织条件、状态条件、删除标记、有效标记。
- 汇总与明细、页面与导出、接口与 SQL 结果一致。
- 信息不足时，只能写 SQL 验证思路，不编造表名字段。

## 金融/交易/策略数据类

- 金额、数量、比例、收益率、成本、盈亏字段精度。
- 汇总与明细是否一致。
- 页面、接口、导出、SQL 四方口径是否一致。
- 交易日、自然日、盘中时间、期初期末时间边界。
- 股票、策略、账户、批次、成交编号等维度是否重复或漏聚合。
- 建仓、加仓、减仓、清仓、撤单、失败、部分成交、重复成交等状态场景。
- 0 值、负值、空值、极大值、科学计数法展示。
- 多买、多卖、一个卖策略对应多个买策略、买卖关系重复统计风险。
- 买入相关数据先取最大值再聚合、卖出相关数据按真实成交聚合的口径风险。
- 权限、账户、组织、策略范围隔离。
- 历史数据、新旧口径、交易日历和非交易日兼容。
- 盘中数据、日终数据、T 日、T-1 日、期初、期末口径一致性。
- 页面汇总、弹窗明细、下钻页面、导出文件之间的数据一致性。
- 批次、股票、策略、账户、成交编号等多维度组合导致的重复统计和漏统计风险。

使用约束：

- 该模板只作为分析辅助，不代表每个金融/交易需求都必须全部生成 TC。
- 只有命中真实需求、diff、SQL 或业务风险时才生成对应测试点。
- 不得凭模板虚构表名、字段名、SQL、接口或页面入口。
- 金融/交易/策略数据类需求中，优先关注核心数据口径、汇总明细一致性、精度、权限隔离和历史兼容。
- 对金额、数量、比例、收益率、成本、盈亏等字段，必须重点扫描精度、单位、四舍五入、空值、0 值、负值和极端值风险。
- 如果需求或 diff 涉及 SQL，必须关注 where、join、group by、distinct、聚合粒度、日期口径、有效标记、权限条件。

## 权限类

- 有权限用户可查看或操作。
- 无权限用户不可查看或操作。
- 不同角色、组织、部门、数据范围隔离。
- 下钻、弹窗、导出、接口直接访问是否继承权限。
- 越权访问、参数篡改、跨组织数据访问。

## 状态和流程类

- 初始状态、处理中状态、终态、异常状态。
- 合法状态流转、非法状态流转、重复提交。
- 状态变更后的列表、详情、汇总、导出和下游同步。
- 历史数据、新旧状态、字典映射和兼容。

## 配置和开关类

- 配置开启、关闭、默认值、灰度范围。
- 配置变更后是否实时生效，是否需要刷新、重新登录或重新查询。
- 配置缺失、异常值、回滚后行为。
- 不同环境、组织、角色下配置差异。

## 日志、审计和通知类

- 关键操作是否记录日志或审计信息。
- 操作人、操作时间、操作对象、变更前后值是否可追踪。
- 通知触发条件、接收人、重复通知、失败补偿。
- 需求未说明日志或通知口径时，只列风险和待确认点，不生成确定预期。
